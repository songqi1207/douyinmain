# 项目报价说明与需求边界清单

## 一、项目目标

本项目目标是替代现有米核/剪映小助手类插件，打通：

`Coze 工作流 -> 自建插件工具 -> 剪映草稿生成/导入`

本阶段的重点不是自研视频渲染器，也不是直接在云端导出最终成品视频，而是先稳定实现以下能力：

- Coze 可以正常调用自建插件
- 工作流中的关键节点可以被本地/自建工具替代
- 支持通过 `draft_key` 或 `draft_id` 生成或同步剪映草稿
- 最终返回本地 Windows 可访问的剪映草稿路径

## 二、一期建议交付目标

- Coze 可正常导入并调用 OpenAPI 插件
- 关键工具链跑通
- 支持以下两种模式中的一种或两种
  - `draft_key -> 本地生成剪映草稿`
  - `draft_id -> 从服务器同步草稿到本地 Windows`
- 返回本地草稿路径，供剪映识别
- 生图能力先接第三方 API，不自研模型
- 至少 1 套主模板完整跑通
- 另外 2 套模板具备迁移基础

## 三、建议报价区间

### 1. 个人开发者报价参考

- 最小可用版：`1.5 万 - 3 万`
- 稳定交付版：`2 万 - 4 万`

### 2. 团队 / 外包报价参考

- 最小可用版：`2.5 万 - 6 万`
- 稳定增强版：`4 万 - 8 万`

## 四、建议工期

- 最小可用版：`2 - 3 周`
- 稳定交付版：`3 - 5 周`

前提是：

- 当前已有部分代码和原型
- 不从零开始研究 Coze、剪映草稿结构和工作流模板
- 生图与语音优先接第三方能力

## 五、工作范围

### 包含范围

- Coze OpenAPI 插件对接
- 工具接口开发与参数兼容
- 剪映草稿生成与写入逻辑
- 服务器草稿打包与本地导入
- 基础前端操作页
- 基础部署、日志、错误处理
- 至少一套模板全链路联调

### 不包含范围

- 自研生图模型
- 自研语音大模型
- 云端自动渲染成最终视频
- 剪映桌面端自动点按钮导出
- 高并发架构改造
- 多平台全量兼容
- 长期运维托管

## 六、需求边界清单

### 1. 生图模型边界

- 本期默认接第三方生图 API
- 只要求返回稳定图片 URL，并能接入工作流
- 不包含自部署 Stable Diffusion、Flux、ComfyUI 集群
- 不包含 LoRA 训练、私有模型微调、复杂风格训练

### 2. 插件工具边界

- 本期重点是“工作流可用”，不是 100% 复刻米核内部实现
- 优先保证核心工具链兼容：
  - `create_draft_from_key`
  - `import_remote_draft`
  - `create_draft`
  - `add_images`
  - `add_captions`
  - `add_audios`
  - `add_keyframes`
  - `add_effects`
- 其他辅助工具以模板能跑通为准
- 不承诺完全复刻第三方插件的全部隐式行为
- Coze 端参数名、数组结构、返回格式，以当前模板实际使用为准

### 3. 草稿与视频边界

- 本期目标是生成剪映草稿，不是直接生成最终视频文件
- 默认支持：
  - 服务器生成草稿
  - 本地 Windows 导入草稿
  - 返回本地草稿路径
- 不包含：
  - 服务器直接调用剪映导出 MP4
  - 云端无人值守视频渲染
  - 远程控制用户桌面剪映完成导出

### 4. 运行环境边界

- 服务端默认 Linux 部署
- 本地导入目标默认 Windows 剪映专业版目录
- 不承诺同时兼容 macOS、多个剪映版本、海外 CapCut 全版本
- 若本机目录、权限、杀软、代理环境异常导致失败，按兼容问题单独处理

### 5. 联调与验收边界

- 一期验收以“主链路跑通”为标准
- 建议验收方式：
  - Coze 内成功调用插件
  - 输入 `draft_key` 或 `draft_id`
  - 本地成功落出草稿目录
  - 剪映能识别该草稿
- 不以“所有历史模板零修改全部通过”作为一期硬性标准
- 超出一期范围的模板细修，建议列入二期

## 七、建议分期

### 一期：最小可用版

- 跑通核心插件链路
- 跑通草稿导入本地
- 跑通至少一个主模板
- 提供基础前端页面
- 提供部署说明

### 二期：稳定增强版

- 三个模板全部适配
- 工具参数进一步补全
- 优化错误提示、日志、重试
- 支持批量任务
- 提供更完整的前端管理页

## 八、建议当面确认的问题

建议和对方当面确认以下事项：

1. 报价是按“工具数量”算，还是按“整条链路跑通”算
2. 一期验收标准到底是什么
3. 模板变动后是否包含二次适配
4. Coze 调试、schema 修正、超时问题是否算在报价内
5. 部署和上线由谁负责
6. 后续维护是按次收费还是按月维护

## 九、建议对外沟通话术

这期不是要做一个完整视频平台，而是先把：

`Coze -> 插件 -> 剪映草稿`

这条链路替换出来。

第一阶段只要求能稳定生成草稿，并把草稿落到本地 Windows 机器上给剪映识别，不要求云端直接出成品视频。

生图先接第三方，预算重点放在以下三块：

- 插件兼容
- 草稿结构
- Coze 联调

## 十、成本结构简表

### 开发成本

- 核心来自人力，不是机器
- 真正耗时的是：
  - Coze 插件兼容
  - 剪映草稿结构适配
  - 模板联调和问题修复

### 运行成本

- 云服务器：`100 - 500 元 / 月`
- 域名与 HTTPS：几十到几百元 / 年
- 图片存储与带宽：`0 - 200 元 / 月`
- 第三方生图 API：按量计费
- 第三方语音 API：按量计费

### 初期月度运行成本参考

- 轻量使用：`300 - 1000 元 / 月`
- 有一定调用量：`1000 - 5000 元 / 月`
