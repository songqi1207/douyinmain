# 业务需求说明书：「每天认识一个神」短视频自动生成工作流

> 版本：v2.0  
> 日期：2026-05-12  
> 状态：开发中

---

## 一、项目概述

### 1.1 产品定位
一键生成抖音/快手短视频的自动化工具。用户输入神明名字，系统自动生成完整的 Coze 工作流 JSON 文件，导入 Coze 执行后自动产出剪映草稿，最终得到可发布的短视频。

### 1.2 目标用户
- 短视频自媒体创作者
- 知识类/文化类账号运营者
- 需要批量生产内容的 MCN 机构

### 1.3 核心价值
- 将传统需要 2-3 小时的视频制作流程压缩到 5 分钟
- 保证视频质量稳定可控
- 降低创作门槛，无需剪辑经验

#### 轮播技术实现
- 基于香烟模板的批处理 add_images + keyframes_infos + add_keyframes 节点结构
- 16张图片通过 `transform_x` 设置不同起始位置（间距1000，-7500到7500）
- 关键帧 values: `起始X|终止X`（不回来），移动距离16000
- 图片 scale=0.6，背景图 scale=1.0
- 背景图节点（bg_layer_001）在轮播图片节点（151678）之前执行，确保在最底层

## 四、技术架构

### 4.1 系统流程

```
用户输入神名 → Flask 后端生成工作流 JSON → 用户下载 txt 文件
→ 导入 Coze 工作流 → Coze 执行（LLM 生文案 + 即梦生图 + TTS 配音）
→ 剪映草稿插件组装视频 → 剪映小助手下载草稿 → 剪映打开编辑/导出
```

### 4.2 核心模块

| 模块 | 文件 | 职责 |
|------|------|------|
| 工作流生成器 | `workflows/god/builder.py` | 组装 Coze 工作流 JSON |
| 时间线合并 | `workflows/god/myth_combine_coze_code.py` | 片头/正文/音效/字幕时间线计算 |
| 轮播图管理 | `workflows/god/intro_images.py` | 神明图片 URL 管理 |
| 文案提示词 | `workflows/god/prompts.py` | LLM system prompt 管理 |
| API 路由 | `routes/api.py` | HTTP 接口 |
| 链接提取 | `crawlers/douyin.py` | 抖音/小红书文案提取 |

### 4.3 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 分镜数量 | 20 | 正文图片/文案段数 |
| 片头每张时长 | 1.5 秒 | 轮播图切换频率 |
| 翻页转场时长 | 0.4 秒 | 图片间转场动画 |
| 翻书音效时长 | 0.3 秒 | 每次翻书声 |
| 敲钟音效时长 | 1.5 秒 | 标题出现时 |
| 入场动画时长 | 0.8 秒 | 正文图片入场 |
| Ken Burns 缩放 | 1.0→1.08 | 静态图缓慢放大 |
| 画布尺寸 | 1920×1080 | 横屏 16:9 |

## 五、竞品参考

### 5.1 参考视频特征
- 开头翻书/轮播效果，音效与画面切换频率契合
- 介绍主角时有抠图 + 文字 + 音效组合
- 图片有简单动作（位移/缩放），不是静止的
- 色彩丰富鲜明，不暗淡
- 文案口语化，像说书/聊天，无 AI 感
- 字体有设计感，不是默认字体

### 5.2 与竞品差距
- ✅ 翻书效果 — 已实现
- ✅ 图片动作 — 已实现
- ✅ 色彩丰富 — 已实现
- ⚠️ 动画名称未验证 — 需在剪映中确认
- ⚠️ 敲钟音效 — 需替换真实音效
- ❌ AI 动态图 — 未实现（P2）

## 六、验收标准

1. 生成的工作流在 Coze 中能正常执行，无报错
2. 剪映草稿能被剪映小助手正常下载和打开
3. 片头有翻书轮播效果 + 音效
4. 正文图片有入场动画，不是静止硬切
5. 字幕有动画效果，不是静态出现
6. 文案读起来像真人说书，无明显 AI 感
7. 画面色彩丰富，无大面积黑灰暗淡
8. 所有图片资源可正常下载（boltp CDN）
